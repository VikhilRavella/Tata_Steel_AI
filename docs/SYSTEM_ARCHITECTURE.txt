SECTION 1
PROJECT OVERVIEW

Purpose
The Tata Steel Industrial AI Platform provides an intelligent, dual-agent workspace designed for industrial maintenance, inventory lookup, work order creation, safety compliance, PDF knowledge extraction, vision diagnostics, and supervisor escalations.

Modules
- Sandbox Agent (Requirement Gathering, PDF Knowledge, General RAG)
- Engineering Agent (Action Execution, Analysis, Reporting, Work Orders)
- Vision Diagnostics (Image Defect Detection)
- Inventory Management (Stock Search, Requests, Transactions)
- Work Order Management (Task Assignment, Status Tracking)
- Supervisor/Manager Portals (Approval Workflows, Dashboards)

Technologies
Frontend: Vanilla HTML, CSS, JavaScript (Web Speech API, MediaRecorder API)
Backend: Python, FastAPI, SQLAlchemy, Uvicorn
Database: SQLite (maintenance.db), ChromaDB (Vector Search)
Ollama Models:
- mistral:latest
- qwen2.5-coder:7b
- qwen2.5vl:latest
- BAAI/bge-small-en-v1.5

SECTION 13
MODEL ARCHITECTURE

mistral:latest
- Used For: Sandbox Agent conversations, PDF summaries, RAG QA, conversational natural language processing.
- Reason: Faster inference, better conversational flow.

qwen2.5-coder:7b
- Used For: Engineering Agent tasks, complex logic, work order generation, Root Cause Analysis (RCA), JSON structured extraction.
- Reason: High logical deduction, coding capabilities, structured output reliability.

qwen2.5vl:latest
- Used For: Vision tasks, image analysis, defect detection.
- Reason: Native image-to-text multimodal support.

BAAI/bge-small-en-v1.5
- Used For: PDF text chunk embedding generation.
- Reason: High efficiency for RAG vector storage in ChromaDB.

SECTION 15
DEMO VIDEO SCRIPT SUPPORT

Step 1
Login: User authenticates via the login page and selects the Engineer profile.

Step 2
Engineering Agent: User navigates to the Engineering Dashboard and starts a session.

Step 3
PDF Upload: User uploads an equipment manual PDF into the Sandbox Agent.

Step 4
Vision Analysis: User uploads an image of a broken valve for the Engineering Agent to diagnose using qwen2.5vl.

Step 5
Inventory Lookup: User asks the agent to check if a specific part is in stock.

Step 6
Spare Part Request: User explicitly requests the part, moving it to pending status.

Step 7
Supervisor Approval: Supervisor logs in, reviews the request on their dashboard, and approves it.

Step 8
Notification: Engineer receives a notification that their part request was approved.

Step 9
Reports: User generates a Root Cause Analysis report based on the completed work.

Step 10
Audit Logging: Manager views the system audit logs via the Manager Portal.
